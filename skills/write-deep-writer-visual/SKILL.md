# Write Deep Writer Visual

深度写作与自动配图智能体 - 将口语化或碎片化内容转化为结构严谨、深度的图文内容。

## 快速开始

```
输入：演讲稿、访谈记录、文章草稿、论文等
输出：5000-10000字深度文章 + 封面图 + 3-8张情境插图
```

**使用方法：**
```
"用 write-deep-writer-visual 处理这个内容"
```

---

## 核心功能

### 📝 深度写作（8层处理系统）

1. **需求识别** - 提取写作原则、受众、场景、目的
2. **Brief构建** - 建立创作基准：人设、受众、立场
3. **预处理** - 处理口语化、重复、碎片化观点
4. **观点提取与聚类** - 提取并合并相似观点
5. **逻辑分析** - 识别因果、并列、递进、对比、总结关系
6. **结构优化** - 增强逻辑、删除重复、保持框架
7. **深度撰写** - 补充背景、数据、案例、扩展观点
8. **质量检查** - 验证所有约束和质量标准

### 🎨 自动配图（视觉生成系统）

1. **封面图生成** - 阶段二后生成匹配的封面图
2. **插图规划** - 识别需要视觉支持的章节
3. **类型×风格选择** - 选择合适的插图类型和风格
4. **图片生成** - 为关键章节生成情境插图

## 工作流程

### 三阶段流程（带进度追踪）

```
阶段一：分析与规划（20%）
├─ 需求识别
├─ Brief文档构建
├─ 预处理
├─ 观点提取与聚类
└─ 逻辑分析

输出：Brief文档 + 观点聚类 + 逻辑图谱
```

```
阶段二：结构设计 + 视觉规划（30%）⭐
├─ 结构优化
└─ 视觉规划 ⭐ 新增
   ├─ 分析配图机会
   ├─ 确定插图位置
   ├─ 选择插图类型
   └─ 选择视觉风格

输出：优化结构 + 视觉规划

↓ 生成封面图（需用户确认）
```

```
阶段三：内容撰写 + 配图生成（50%）⭐
├─ 深度撰写
├─ 插图生成 ⭐ 新增
│  ├─ 根据视觉规划生成
│  ├─ 使用图片生成API
│  └─ 插入到文章相应位置
└─ 质量检查

输出：完整图文内容
```

## 视觉配置指南

### 封面图配置

| 内容类型 | 推荐类型 | 推荐色调 | 推荐渲染 |
|----------|----------|----------|----------|
| 学术论文 | conceptual | cool | flat-vector |
| 商业文档 | hero | elegant | digital |
| 通俗教程 | scene | warm | hand-drawn |
| 科技内容 | metaphor | dark | digital |
| 生活方式 | minimal | pastel | painterly |

### 插图类型

| 类型 | 适用场景 | 示例 |
|------|----------|------|
| `infographic` | 数据、指标、技术 | 统计数据、对比分析 |
| `scene` | 叙事、故事 | 案例研究、场景描述 |
| `flowchart` | 教程、工作流 | 流程说明、步骤演示 |
| `comparison` | 对比分析 | 优劣势、前后对比 |
| `framework` | 方法论、模型 | 理论框架、体系结构 |
| `timeline` | 历史、进展 | 发展阶段、时间线 |

### 插图密度

| 密度 | 图片数量 | 适用场景 |
|------|----------|----------|
| `minimal` | 1-2张 | 简短文章，以文字为主 |
| `balanced` | 3-5张 | 标准文章 |
| `per-section` | 5-8张 | 长文、教程 |
| `rich` | 8+张 | 视觉化内容 |

### 视觉风格

| 风格 | 特点 | 适用场景 |
|------|------|----------|
| `notion` | 简洁专业 | 学术、商业 |
| `warm` | 温暖亲和 | 教育、生活 |
| `minimal` | 极简主义 | 技术、设计 |
| `blueprint` | 蓝图风格 | 工程、架构 |
| `watercolor` | 水彩风格 | 文艺、创意 |
| `elegant` | 优雅学术 | 论文、报告 |

## 约束条件

### 写作约束（来自 deep-writer）

1. **Structure不可重组** - 结构框架确定后，严格遵循
2. **Viewpoints必须合并** - 所有重复观点必须合并
3. **Logic必须清晰** - 合并优化后要有清晰逻辑链
4. **Depth要有度** - 补充内容必须高度相关
5. **Expression要准确** - 不能歪曲原始核心观点
6. **Brief是基准** - 所有步骤必须符合Brief要求
7. **Stages必须确认** - 每阶段后必须等待用户确认

### 视觉约束（新增）

8. **Images要相关** - 所有插图必须直接支持内容理解
9. **风格要一致** - 保持统一的视觉风格
10. **数量要适度** - 不要过度配图，平衡文字与视觉
11. **Cover要精准** - 封面图必须反映核心主题
12. **确认要完成** - 封面生成和图片密度必须获得用户确认

## 技术集成

### 与 baoyu 技能的集成

#### 视觉规划（阶段二）
```bash
# 使用 baoyu-article-illustrator 分析内容
SKILL_DIR=~/.claude/skills/baoyu-article-illustrator

# 询问用户视觉偏好
# - 插图密度（minimal/balanced/per-section/rich）
# - 首选类型（auto/infographic/scene/flowchart等）
# - 首选风格（auto/notion/warm/minimal等）
```

#### 封面生成（阶段二后）
```bash
# 使用 baoyu-cover-image
SKILL_DIR=~/.claude/skills/baoyu-cover-image
PROMPT_FILE=outline.md  # Brief + 结构
OUTPUT=cover.png

# 基于内容分析自动选择维度
# 生成前询问用户确认
```

#### 插图生成（阶段三）
```bash
# 使用 baoyu-image-gen
SKILL_DIR=~/.claude/skills/baoyu-image-gen

# 对规划中的每张插图：
# - 基于章节内容创建prompt
# - 使用选定的类型/风格生成
# - 插入到文章相应位置
```

## 常见问题

**Q: 图片质量不理想？**
A: 可以提高分辨率参数（--size 2048x1024）或更换API服务

**Q: 如何调整图片风格？**
A: 在调用时指定：风格用minimal，色调用warm

**Q: 可以只生成图片不改写内容吗？**
A: 建议使用 baoyu-article-illustrator 专门配图

**Q: 支持哪些语言？**
A: 支持中文、英文等多种语言

**Q: 生成时间需要多久？**
A: 3000字+3张图约5-10分钟，10000字+6张图约15-20分钟
